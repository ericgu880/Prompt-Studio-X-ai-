# 1. 产品背景

随着 Midjourney、Nano Banana、Image 2、Seedream、Seedance、可灵等图像和视频生成模型的使用频率提升，创作者会积累大量 Prompt、参考图、生成图、版本变体和参数组合。

目前用户常见的管理方式包括：

- 把 Prompt 存在 Notion、备忘录、飞书文档或聊天记录里；
- 把图片存在 Finder、Eagle、相册、网盘或项目文件夹里；
- 把模型参数散落在截图、文件名、导出记录或生成平台历史中；
- 每次复用 Prompt 时，需要重新翻找、复制、改写和测试。

这会导致几个问题：

1. **Prompt 与生成结果脱节**：只看到图，不知道用什么 Prompt；只看到 Prompt，不知道最终效果。
2. **版本迭代不可追踪**：V1、V2、V3 的差异靠记忆，难以复盘。
3. **模型参数难以复用**：比例、风格、负面提示词、参考图、种子等信息不统一。
4. **本地素材查找效率低**：大量图片靠文件名或文件夹管理，缺少标签、模型、风格维度。
5. **跨模型 Prompt 适配成本高**：同一个创意可能要改写成 Midjourney、Seedream、Image 2、可灵等不同模型的格式。

因此，PromptStudio 的目标是做成一个 **面向 AI 创作者的本地优先 Prompt 工作台**，让用户可以把图片、视频、Prompt、参考图、参数和版本统一管理起来。

# 2. 产品定位

PromptStudio 不是单纯的图片素材库，也不是纯文本 Prompt 笔记工具，而是：

> 一个 macOS 本地优先的 AI 创作资产管理客户端，用来管理 Prompt、生成结果、模型参数、参考图、标签和版本历史。

## 2.1 一句话定位

**PromptStudio：把你的 AI Prompt、生成图和创作版本统一管理起来。**

## 2.2 产品边界

PromptStudio 的核心应围绕“管理、复用、迭代”展开。

### 应该做

- 管理 Prompt；
- 管理生成图片与视频；
- 绑定图片与 Prompt；
- 管理参考图、标签、模型、参数；
- 支持版本记录与复用；
- 支持本地导入、搜索、筛选、导出；
- 后续可扩展 AI 改写与生成变体。

### MVP 不建议重点做

- 不做完整生图平台；
- 不强制接入所有模型 API；
- 不做复杂团队协作；
- 不做云端同步；
- 不做浏览器插件；
- 不做 Windows 客户端；
- 不做复杂项目管理系统。

# 3. 产品目标

## 3.1 MVP 目标

MVP 版本的目标是完成一个稳定、易用、可上线的 macOS 本地客户端，支持用户完成以下核心闭环：

1. 导入图片 / 视频 / Prompt；
2. 给素材绑定 Prompt、模型、标签、参数；
3. 在瀑布流中浏览和搜索；
4. 在右侧详情面板中查看和编辑 Prompt；
5. 支持 Prompt 复制、版本管理、参考图管理；
6. 支持本地数据备份和导出。

## 3.2 产品成功指标

MVP 可通过以下指标判断是否成功：

| 指标 | 目标 |
|---|---|
| 首次导入完成率 | 新用户首次启动后 10 分钟内完成一次导入 |
| Prompt 复制频次 | 用户日常高频使用“复制提示词” |
| 搜索使用率 | 用户能通过搜索和标签找到目标素材 |
| 资料库留存 | 用户持续向本地资料库添加内容 |
| 版本使用率 | 用户在同一 Prompt 下保存多个版本 |
| 导出成功率 | 用户可以稳定导出图片或 Prompt 数据 |

## 3.3 非目标

MVP 阶段暂不把以下能力作为核心目标：

- 不直接内置完整 AI 生成工作流；
- 不做多人团队协作；
- 不做实时云同步；
- 不做模板商城；
- 不做多端客户端；
- 不做复杂权限管理。

其中，“生成变体”可以作为轻量功能进入 MVP 或 v1.1，但建议先设计为 **Prompt 文本变体生成**，而不是直接连接所有图像生成模型。

# 4. 核心价值

## 4.1 对用户的价值

1. **找得快**：通过全局搜索、标签、模型、比例、分类快速定位 Prompt 和图片。
2. **看得懂**：每张图都能看到标题、模型、比例、标签、Prompt、负面提示词、参考图和版本。
3. **复用快**：一键复制 Prompt，一键复制某个历史版本，一键生成变体。
4. **管理清晰**：图片 Prompt、视频 Prompt、收藏、最近使用、标签、回收站结构清晰。
5. **本地安全**：数据优先存储在用户本地，适合创作者管理个人素材和商业项目。

## 4.2 对产品的价值

1. 建立创作者长期使用的本地资产库；
2. 通过高频 Prompt 复用形成用户黏性；
3. 后续可扩展为付费版、模型 API 工作流、云同步、团队库、模板市场。

# 5. 目标用户与典型场景

## 5.1 用户类型

| 用户类型 | 使用场景 | 核心需求 |
|---|---|---|
| AI 绘图创作者 | 管理 Midjourney、Image 2、Nano Banana 等生成图 | 快速找到图对应的 Prompt，复用和改写 |
| 视频生成创作者 | 管理 Seedance、可灵等视频 Prompt | 区分图片 Prompt 和视频 Prompt，记录镜头、风格、运动描述 |
| 设计师 / 插画师 | 管理风格参考图、构图参考图、色彩参考 | 建立个人风格库和灵感库 |
| 自媒体 / 商业创作者 | 批量管理封面、海报、分镜、角色图 | 快速复用高质量 Prompt 模板 |
| Prompt 工程爱好者 | 研究不同模型的提示词表现 | 版本对比、参数归档、模型适配 |

## 5.2 用户画像 A：AI 插画创作者

**痛点：**  
每天生成几十张图，Prompt 散落在聊天记录里。好看的图过几天找不到对应 Prompt。

**使用流程：**

1. 把生成图拖入 PromptStudio；
2. 填写标题、模型、Prompt、标签；
3. 添加参考图和风格色板；
4. 后续通过“风景 / 人物 / 写实 / Midjourney”筛选；
5. 复制 Prompt 后去模型平台继续生成。

## 5.3 用户画像 B：短视频 AI 创作者

**痛点：**  
视频 Prompt 需要记录镜头运动、时长、风格、人物动作、负面词。普通图库无法管理。

**使用流程：**

1. 在左侧进入“视频 Prompt”；
2. 新建一个 Seedance 或可灵 Prompt；
3. 写入镜头描述、动作描述、风格描述；
4. 导入生成视频和封面图；
5. 后续按项目或模型筛选复用。

## 5.4 用户画像 C：商业设计师

**痛点：**  
项目多、客户多、素材多，需要按项目和风格管理 Prompt。

**使用流程：**

1. 建立项目文件夹；
2. 导入客户项目相关图片；
3. 使用标签标记“海报 / 科技 / 写实 / 电商 / 角色”；
4. 对 Prompt 进行版本迭代；
5. 导出图片和 Prompt 交付给团队或客户。

# 6. 核心用户流程

## 6.1 新建 Prompt 流程

```text
点击「新建 Prompt」
→ 选择类型：图片 Prompt / 视频 Prompt
→ 填写标题
→ 选择模型
→ 输入 Prompt
→ 输入负面提示词，可选
→ 添加标签
→ 添加参考图，可选
→ 保存
→ 在瀑布流中生成一张 Prompt 卡片
```

## 6.2 导入图片并绑定 Prompt 流程

```text
拖入图片
→ 系统生成缩略图
→ 用户填写或粘贴 Prompt
→ 选择模型、比例、分类、标签
→ 保存
→ 图片出现在对应资源库
→ 右侧详情面板可查看完整信息
```

## 6.3 复用 Prompt 流程

```text
搜索或筛选目标图片
→ 点击卡片
→ 右侧查看 Prompt
→ 点击「复制提示词」
→ 粘贴到外部模型平台
```

## 6.4 版本管理流程

```text
打开某个 Prompt
→ 点击编辑
→ 修改 Prompt 或参数
→ 保存为新版本
→ 系统生成 V1.1 / V1.2 / V2.0
→ 可随时切换、复制、恢复历史版本
```

## 6.5 参考图管理流程

```text
打开 Prompt 详情
→ 点击参考图区域 +
→ 添加风格参考图 / 构图参考图 / 角色参考图 / 色彩参考
→ 系统展示缩略图和类型
→ 后续复制 Prompt 时可同时查看参考图
```

# 8. 页面与功能需求

## 8.1 主界面

### 页面目标

主界面用于完成 Prompt 和素材的浏览、搜索、筛选、查看、编辑和复用。

### 页面结构

页面采用三栏结构：

```text
左侧导航栏 + 中间瀑布流内容区 + 右侧详情面板
```

### 功能要求

| 模块 | 功能 | 优先级 |
|---|---|---|
| 左侧导航 | 资源库、图片 Prompt、视频 Prompt、收藏、最近使用、标签、回收站 | P0 |
| 顶部搜索 | 搜索名称、Prompt、标签、描述、模型 | P0 |
| 模型筛选 | 按模型筛选内容 | P0 |
| 瀑布流图库 | 展示图片 / 视频 / Prompt 卡片 | P0 |
| 卡片快捷操作 | 编辑、复制、收藏、版本、更多 | P0 |
| 右侧详情 | 展示选中项完整信息 | P0 |
| 视图切换 | 网格视图 / 列表视图 | P1 |
| 批量选择 | 多选、批量打标签、批量删除 | P1 |

---

## 8.2 左侧导航栏

### 设计目标

左侧导航栏应当解决“资源分类混乱”的问题，让用户清楚知道自己当前在哪个资源库、哪个项目、哪个模型或哪个标签下。

### 功能需求

1. 支持资源库分组；
2. 支持图片 Prompt 和视频 Prompt 分开管理；
3. 支持项目 / 文件夹层级；
4. 支持收藏入口；
5. 支持最近使用；
6. 支持标签快捷筛选；
7. 支持回收站；
8. 支持新增文件夹 / 项目；
9. 支持重命名、删除、拖拽排序。

### 交互要求

- 当前选中项需要有明显高亮；
- 支持折叠和展开；
- 文件夹右侧显示数量；
- 最近使用显示缩略图和时间；
- 标签显示数量；
- 回收站显示被删除数量。

```text
缩略图
```

# 14. 本地数据与文件管理

## 14.1 数据存储原则

PromptStudio 应采用 **本地优先** 策略：

1. 用户数据默认保存在本地；
2. 图片、视频、Prompt、标签、版本信息均可离线访问；
3. 不经用户授权，不上传任何图片、Prompt 或 API Key；
4. 后续云同步作为独立能力扩展。

## 14.2 本地资料库

建议用户首次启动时选择资料库存储位置：

```text
~/Documents/PromptStudio Library
```

或使用 App 默认容器目录。

资料库结构建议：

```text
PromptStudio Library
├─ assets
│  ├─ images
│  ├─ videos
│  └─ references
├─ thumbnails
├─ database
│  └─ promptstudio.sqlite
├─ exports
└─ backups
```

## 14.3 文件引用策略

提供两种模式：

| 模式 | 说明 |
|---|---|
| 复制到资料库 | 导入时复制一份文件到 PromptStudio Library |
| 引用原文件 | 只记录原文件路径，不复制文件 |

MVP 建议默认使用“复制到资料库”，避免用户移动原文件后丢失。

# 20. 上线与分发策略

macOS 客户端有两条主要分发路线：**Mac App Store** 和 **官网 / 独立渠道分发**。

## 20.1 路线 A：Mac App Store

优点：

- 用户信任度高；
- 更新由 App Store 管理；
- 可使用 App Store 付费和内购；
- 更容易被普通用户接受。

限制：

- Mac App Store 应用需要适当沙盒化；
- 需要使用 Xcode / App Store Connect 流程提交；
- 不适合使用第三方安装器；
- 更新需要通过 Mac App Store；
- 应使用公开 API，并避免不符合审核要求的行为。

对 PromptStudio 的影响：

1. 需要谨慎设计文件访问权限；
2. 资料库目录建议放在 App Container 内，或通过用户授权选择目录；
3. 如果要访问任意 Finder 路径，需要符合沙盒文件访问规则；
4. 如果上架 App Store，自动更新不能使用第三方更新器；
5. 如果做付费订阅或内购，需要走 App Store 机制。

## 20.2 路线 B：官网分发 / Developer ID

优点：

- 上线节奏更灵活；
- 可使用自己的授权、订阅、更新机制；
- 对本地文件访问限制相对更灵活；
- 适合先做商业验证。

要求：

- 需要 Apple Developer Program 会员；
- 需要 Developer ID 证书；
- 应提交 Apple notarization；
- 需要自行管理下载安装、软件更新、支付、用户支持。

对 PromptStudio 的影响：

1. 可以更快发布 MVP；
2. 更适合频繁调整资料库和导入导出逻辑；
3. 需要自建更新机制，例如 Sparkle 或自研更新服务；
4. 需要做好签名、公证、下载页、版本发布说明。

## 20.3 推荐策略

MVP 建议采用：

```text
第一阶段：官网分发 + Developer ID + Notarization
第二阶段：稳定后评估 Mac App Store 上架
```

原因：

1. PromptStudio 是本地素材管理工具，早期会频繁调整文件管理、导入导出和资料库逻辑；
2. 官网分发更方便快速迭代；
3. 等核心能力稳定后，再根据商业模式决定是否上架 Mac App Store；
4. 如果未来做 App Store 版本，需要提前评估沙盒权限、内购、更新机制和文件访问策略。

2. 不阻塞主界面；
3. 支持缩略图缓存；
4. 原图不压缩；
5. 视频生成封面帧。

## 22.4 搜索

MVP 搜索要求：

1. 支持标题搜索；
2. 支持 Prompt 正文搜索；
3. 支持标签搜索；
4. 支持模型搜索；
5. 支持文件名搜索。

P1 可增加全文索引和搜索高亮。

# 30. 验收标准

## 30.1 新建 Prompt

- 用户点击“新建 Prompt”后可以创建一条记录；
- 必填字段为空时不能保存；
- 保存后中间图库出现新卡片；
- 右侧详情显示新 Prompt；
- 数据重启后仍存在。

## 30.2 导入图片

- 用户可拖拽 PNG / JPG 到窗口；
- 系统生成缩略图；
- 图片出现在瀑布流；
- 右侧显示格式、尺寸、大小；
- 文件保存到本地资料库。

## 30.3 搜索

- 输入标题关键词可搜索到对应卡片；
- 输入 Prompt 关键词可搜索到对应卡片；
- 输入标签可搜索到对应卡片；
- 清空搜索后恢复全部内容。

## 30.4 复制 Prompt

- 点击“复制提示词”后，Prompt 写入剪贴板；
- 用户可粘贴到外部应用；
- 如果 Prompt 为空，按钮不可用或提示用户。

## 30.5 版本管理

- 用户修改 Prompt 后可保存为新版本；
- 右侧显示版本按钮；
- 点击版本可查看对应 Prompt；
- 可以复制历史版本 Prompt。

## 30.6 回收站

- 删除素材后不立即永久删除；
- 回收站中可以恢复；
- 永久删除前必须二次确认。

# 31. 风险与解决方案

| 风险 | 影响 | 解决方案 |
|---|---|---|
| 本地文件路径丢失 | 图片无法打开 | 默认复制到资料库 |
| 大量图片导致卡顿 | 影响体验 | 缩略图缓存、懒加载、分页加载 |
| Prompt 数据结构过于复杂 | MVP 延期 | 先做通用字段，模型参数 P1 扩展 |
| App Store 沙盒限制 | 上架困难 | MVP 先官网分发，后续单独适配 |
| 用户不知道如何开始 | 首次体验差 | 做空状态引导和示例库 |
| 生成变体涉及联网 | 隐私顾虑 | 默认关闭，明确授权 |
| 商业模式不清晰 | 影响架构 | 尽早确定免费版 / Pro 版边界 |

# 32. 关键待确认问题

## 32.1 产品范围

**待确认：** MVP 是否只做 Prompt 管理，还是必须内置 AI 生成能力？

建议：  
MVP 先不做完整生图，只做 Prompt 管理和文本变体生成。这样更容易上线，也更符合本地客户端定位。

## 32.2 分发方式

**待确认：** 首发是官网分发，还是直接 Mac App Store？

建议：  
首发官网分发 + Developer ID + notarization。后续稳定后再评估 Mac App Store。

## 32.3 商业模式

**待确认：** 是免费工具、一次性买断、订阅制，还是免费 + Pro？

建议：  
先做免费试用 + Pro 买断或订阅。创作者工具比较适合 Pro 版。

## 32.4 是否支持视频

**待确认：** Seedance、可灵等视频 Prompt 是否必须进入 v1.0？

建议：  
v1.0 可以支持“视频 Prompt 文本管理”，但视频文件导入可以放到 v1.1。

## 32.5 是否支持云同步

**待确认：** 是否要做 iCloud / 自建云同步？

建议：  
v1.0 不做云同步，先把本地资料库稳定性做好。

# 33. PRD 总结

PromptStudio 的 MVP 应该围绕一个核心闭环展开：

```text
导入素材
→ 绑定 Prompt
→ 添加模型 / 标签 / 参考图
→ 浏览和搜索
→ 查看详情
→ 复制复用
→ 版本迭代
→ 本地导出
```

当前页面的三栏布局方向是正确的。真正需要强化的是：

1. 左侧资源结构更清晰；
2. 中间卡片从“图片墙”升级为“Prompt 资产卡片”；
3. 右侧详情面板升级为“Prompt 编辑器”；
4. 增加版本、标签、模型参数和参考图管理；
5. 采用本地优先策略，先做稳定的 macOS 客户端。
